UMLsec Static Secure Links Check

Overview

This check analyzes UML deployment diagrams marked with the UMLsec stereotype «secure links» (cf. "Secure Systems Development with UML", p. 59 ff.). For each dependency between deployed artifacts that is marked with security requirements, the corresponding communication path's linktype needs to provide enough security to uphold the given security requirements. The security of the linktype is determined in conjunction with the adversary type affecting the communication path.

Usage

Annotate a model containing a deployment diagram with the UMLsec stereotypes used by the «secure links» property. Afterwards create a new CARiSMA analysis on the model. Add the Static Secure Links Check to the used checks and click "Run" to start the analysis.

The Analysis Results View displays whether the check was successful. If requirements are violated, informations about the location and type of violation are also displayed in the Analysis Results View.

If you don't declare an adversary type when using the «secure links» stereotype, the check uses the default adversary for its analysis. The following tables show the various adversary types and the capabilities of each adversary for every available linktype.
